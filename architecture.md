# Architecture

The Smo.OS ecosystem is structured in layers.

User
↓
Smo.OS / PLOS (memory layer)
↓
Overlord Runtime (cognitive orchestration)
↓
Execution Engine (Temporal or other)
↓
Agents
↓
External tools

## Key design principles

- memory sovereignty
- replaceable execution engines
- agent specialization
- capability-based security
